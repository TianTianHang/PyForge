# debug-system Specification

## Purpose

提供全局调试系统，允许通过环境变量控制调试信息的输出，便于开发环境调试和生产环境日志控制。

## Requirements

### Requirement: 环境变量控制调试开关

系统 SHALL 支持 `PYFORGE_DEBUG` 环境变量来控制调试信息的输出。

#### Scenario: 开启调试模式

- **WHEN** 设置环境变量 `PYFORGE_DEBUG=1`（任意非空值）
- **THEN** 系统输出所有调试信息到 stderr

#### Scenario: 关闭调试模式

- **WHEN** 不设置 `PYFORGE_DEBUG` 环境变量或设置为空
- **THEN** 系统不输出任何调试信息

### Requirement: uv 命令执行调试

系统 SHALL 在调试模式下输出 uv 命令执行的详细信息。

#### Scenario: 成功执行 uv 命令

- **WHEN** 调试模式开启且成功执行 `uv venv` 命令
- **THEN** 系统输出执行的完整命令（包括 `--config-file` 参数）
- **AND** 系统输出命令退出码
- **AND** 系统输出标准输出内容
- **AND** 如果有标准错误，系统输出标准错误内容

#### Scenario: uv 命令执行失败

- **WHEN** 调试模式开启且 `uv` 命令执行失败
- **THEN** 系统输出执行失败的错误信息
- **AND** 系统输出失败原因（sidecar 获取失败、命令执行失败等）

#### Scenario: 关闭调试模式

- **WHEN** 调试模式关闭且执行任何 uv 命令
- **THEN** 系统不输出任何调试信息
- **AND** 命令正常执行和返回结果

### Requirement: 环境创建流程调试

系统 SHALL 在调试模式下输出环境创建的详细流程信息。

#### Scenario: 开始创建环境

- **WHEN** 调试模式开启且开始创建环境
- **THEN** 系统输出环境基本信息：
  - 环境名称
  - 环境ID
  - Python 版本
  - 环境目录路径
  - Python 可执行文件路径
  - 待安装的包列表

#### Scenario: 创建虚拟环境

- **WHEN** 调试模式开启且 Python 不存在，需要创建虚拟环境
- **THEN** 系统输出"开始创建虚拟环境"消息
- **AND** 系统输出"环境目录已创建"消息及路径
- **AND** 系统输出"执行 uv venv 命令"消息
- **AND** 系统输出"虚拟环境创建成功"消息

#### Scenario: 虚拟环境创建失败

- **WHEN** 调试模式开启且虚拟环境创建失败
- **THEN** 系统输出"创建虚拟环境失败"消息
- **AND** 系统输出详细的错误信息（stderr）

#### Scenario: Python 已存在

- **WHEN** 调试模式开启且 Python 可执行文件已存在
- **THEN** 系统输出"Python 已存在，跳过虚拟环境创建"消息

#### Scenario: 安装包

- **WHEN** 调试模式开启且开始安装包
- **THEN** 系统输出"开始安装包"消息
- **AND** 系统输出目标 Python 路径
- **AND** 系统输出"执行 uv pip install 命令"消息
- **AND** 系统输出"包安装成功"消息

#### Scenario: 安装包失败

- **WHEN** 调试模式开启且包安装失败
- **THEN** 系统输出"安装包失败"消息
- **AND** 系统输出详细的错误信息（stderr）

#### Scenario: 无需安装包

- **WHEN** 调试模式开启且包列表为空
- **THEN** 系统输出"无需安装包"消息

### Requirement: Jupyter 内核注册调试

系统 SHALL 在调试模式下输出 Jupyter 内核注册的详细流程信息。

#### Scenario: 注册内核

- **WHEN** 调试模式开启且开始注册 Jupyter 内核
- **THEN** 系统输出"开始注册 Jupyter 内核"消息
- **AND** 系统输出内核名称（`pyforge-{env_id}`）
- **AND** 系统输出"Jupyter 内核注册成功"消息

#### Scenario: 绑定内核到项目

- **WHEN** 调试模式开启且存在 project_id
- **THEN** 系统输出"绑定内核到项目"消息及项目ID
- **AND** 如果绑定成功，系统输出"内核绑定到项目成功"消息
- **AND** 如果绑定失败，系统输出警告消息及错误原因

#### Scenario: 保存元数据

- **WHEN** 调试模式开启且环境创建完成
- **THEN** 系统输出"保存环境元数据"消息
- **AND** 系统输出"环境元数据保存成功"消息

#### Scenario: 环境创建完成

- **WHEN** 调试模式开启且所有步骤完成
- **THEN** 系统输出"环境创建完成"消息及环境ID

### Requirement: 调试信息格式

系统 SHALL 使用统一的调试信息格式。

#### Scenario: 调试信息前缀

- **WHEN** 系统输出调试信息
- **THEN** 每条调试信息使用 emoji 前缀表示类型：
  - 🔧 工具/命令执行
  - 📊 状态信息
  - 📤 输出信息
  - ⚠️ 警告信息
  - ❌ 错误信息
  - ✅ 成功信息
  - 📁 目录/文件操作
  - 📦 包管理操作
  - 🔌 内核/连接操作
  - 🔗 绑定操作
  - 💾 保存操作
  - 🎉 完成信息
  - ℹ️ 一般信息

#### Scenario: 模块标识

- **WHEN** 系统输出调试信息
- **THEN** 每条调试信息包含模块标识：
  - `[UV]` - uv 命令相关
  - `[ENV]` - 环境管理相关

### Requirement: 性能要求

系统 SHALL 确保调试功能不影响正常性能。

#### Scenario: 调试模式关闭时的性能

- **WHEN** 调试模式关闭
- **THEN** 系统不执行任何字符串格式化操作
- **AND** 调试宏在编译期优化为零开销

#### Scenario: 调试模式开启时的性能

- **WHEN** 调试模式开启
- **THEN** 系统仅在需要时检查环境变量（启动时）
- **AND** 调试信息输出不影响业务逻辑执行时间
