## Context

PyForge 已经在提案 1 中实现了全局配置系统后端（配置模型、加载/保存、uv.toml 生成）。现在需要实现 Settings 用户界面和数据目录迁移功能。

当前状态：
- 后端配置系统已完成（提案 1）
- 前端没有 Settings 页面
- 用户无法通过界面修改配置
- 用户无法迁移数据目录

## Goals / Non-Goals

**Goals:**
- 实现 Settings 主页面，包含源设置、路径设置、关于 Tab
- 实现 PyPI 镜像源选择器（官方、清华、阿里、自定义）
- 实现 Python 安装源选择器（官方、腾讯源、agentsmirror、自定义）
- 实现 Python 下载策略选择器（自动、手动、禁止）
- 实现数据目录选择器和目录可写性检测
- 实现数据目录迁移功能（文件移动）
- 迁移后自动重启应用
- 在 App.tsx 中添加设置入口

**Non-Goals:**
- 配置文件热更新（需要重启应用）
- 配置版本管理
- 配置备份功能
- 多用户配置隔离

## Decisions

### 1. Settings 页面结构：Tab 容器

**选择**: 使用 Tab 容器组织设置项
**理由**:
- 符合用户习惯，设置项分类清晰
- 易于扩展新的设置类别
- 代码结构清晰，每个 Tab 对应一个组件

**替代方案**:
- 单页面滚动：设置项太多时不方便
- 多级菜单：复杂度高

### 2. 源配置：预设选项 + 自定义

**选择**: 提供预设选项和自定义输入
**理由**:
- 预设选项方便常用配置
- 自定义输入满足特殊需求
- 减少用户输入错误

**替代方案**:
- 纯自定义输入：用户需要知道完整 URL
- 纯预设选项：灵活性不足

### 3. 数据目录迁移：文件移动

**选择**: 使用文件移动而非复制
**理由**:
- 速度快，不需要等待复制完成
- 节省磁盘空间
- 用户已确认使用文件移动

**替代方案**:
- 文件复制：安全但慢
- 符号链接：复杂，跨平台支持差

### 4. 目录可写性检测：测试文件方式

**选择**: 创建测试文件验证目录可写性
**理由**:
- 可靠性高，直接测试写入能力
- 跨平台兼容
- 可以捕获具体错误信息

**替代方案**:
- 权限检查：不可靠（权限正确但可能被其他原因阻止）
- 尝试创建目录：不够全面

### 5. 迁移后处理：自动重启应用

**选择**: 迁移完成后自动重启应用
**理由**:
- 用户已确认
- 确保配置立即生效
- 用户体验完整

**替代方案**:
- 提示用户手动重启：需要用户操作
- 动态刷新：复杂度高，可能有副作用

### 6. Settings 入口位置：App.tsx Header

**选择**: 在 App.tsx 中添加设置按钮
**理由**:
- 全局可访问
- 符合用户习惯
- 不需要修改现有页面结构

**替代方案**:
- 导航栏：需要修改导航结构
- 右键菜单：不直观

## Risks / Trade-offs

### 风险 1: 迁移过程中断电

**风险**: 迁移过程中断电可能导致数据丢失
**缓解**:
- 迁移前检查磁盘空间
- 优先移动小文件
- 提供错误恢复机制

### 风险 2: 目录可写性检测失败

**风险**: 检测通过但实际写入失败
**缓解**:
- 使用真实文件操作检测
- 捕获具体错误信息
- 提供重试机制

### 风险 3: 配置格式错误

**风险**: 用户输入无效配置
**缓解**:
- 前端验证输入格式
- 后端验证配置有效性
- 提供错误提示

### 权衡 1: 配置更新延迟

**权衡**: 配置更新需要重启应用
**影响**: 用户体验不连续
**接受**: 配置更新频率低，可接受

### 权衡 2: 迁移性能

**权衡**: 文件移动可能需要时间
**影响**: 大数据目录迁移慢
**接受**: 用户通常数据量不大，可接受

## Migration Plan

### 部署步骤

1. 创建配置类型定义（types/config.ts）
2. 创建配置管理 Hook（hooks/useSettings.ts）
3. 创建 Settings 主页面（components/Settings/Settings.tsx）
4. 创建源设置 Tab（components/Settings/SourceSettings.tsx）
5. 创建路径设置 Tab（components/Settings/PathSettings.tsx）
6. 创建关于 Tab（components/Settings/AboutTab.tsx）
7. 创建目录可写性检测逻辑（infrastructure/dir_validator.rs）
8. 创建数据迁移逻辑（infrastructure/data_migration.rs）
9. 修改 App.tsx 添加设置入口
10. 修改 Tauri 命令接口添加迁移命令

### 回滚策略

- 保留现有代码结构
- 新功能可选启用
- 配置系统独立运行

### 测试计划

1. 单元测试：配置类型序列化/反序列化
2. 集成测试：Settings 页面渲染
3. 端到端测试：配置更新流程
4. 端到端测试：数据目录迁移流程
5. 回归测试：现有功能不受影响
